
## Что меняется

Сейчас на странице есть два отдельных блока с формами:
- `QuickSelectSection` — параметры подбора (тип задачи, площадь, человек, высота, город), отправка через `alert()`, никуда не сохраняется.
- `LeadFormSection` — контактная форма (имя, телефон, город) + товары из корзины, сохраняется в таблицу `orders` и отправляется в Telegram.

Объединим их в один блок с заголовком **«Подбор и расчет оборудования»** и единой кнопкой **«Получить расчет»**. Все поля будут сохраняться одной заявкой в админке.

---

## 1. Новый объединённый компонент

Создам `src/components/SelectAndQuoteSection.tsx` (объединяет логику обоих текущих блоков). Старые компоненты `QuickSelectSection.tsx` и `LeadFormSection.tsx` удалю.

**Поля формы (в одной аккуратной сетке, дизайн в стиле текущих блоков):**

Параметры объекта:
- Тип задачи (select) — как в QuickSelect
- Площадь, м² (select) — как в QuickSelect
- Количество человек (select) — как в QuickSelect
- Высота объекта (select) — как в QuickSelect
- **Срок аренды (select)** — новое поле: «День», «Неделя», «Месяц», «Более месяца»
- Город (input)

Контакты:
- Имя (input)
- Телефон (input)

Дополнительно:
- **Комментарий (textarea)** — новое поле

Под формой — сводка корзины (как в текущем `LeadFormSection`), если в корзине есть товары.

**Кнопка:** «Получить расчет» (одна, в стиле текущей кнопки заявки).

**Заголовок секции:**
- Эйбраун: «Подбор и расчет»
- H2: «Подбор и расчет оборудования»
- Подзаголовок: «Оставьте параметры объекта — мы подберем комплект и рассчитаем стоимость аренды»

Сохраню те же отступы (`py-20 md:py-28`), фон (`bg-muted/40 border-y`), контейнер (`max-w-3xl`), стиль карточки и ссылку на политику конфиденциальности.

`id` секции оставлю `lead-form` — чтобы все существующие якорные ссылки/кнопки CTA («Оставить заявку» / «Рассчитать аренду») продолжали работать без изменений.

## 2. Сохранение в админку (одна заявка)

Расширю таблицу `orders` через миграцию — добавлю новые **nullable** колонки, чтобы старые заявки не сломались:

```sql
ALTER TABLE public.orders
  ADD COLUMN task_type   text,
  ADD COLUMN area        text,
  ADD COLUMN people      text,
  ADD COLUMN height      text,
  ADD COLUMN rental_term text,
  ADD COLUMN comment     text;
```

RLS уже настроена корректно (anon может INSERT, admin читает) — менять не нужно.

Обновлю `src/contexts/OrdersContext.tsx`:
- расширю тип `Order` новыми опциональными полями;
- `addOrder` будет принимать и сохранять их;
- передам их в payload `notify-telegram`.

Форма отправит **одну** заявку в `orders` со всеми полями + товарами корзины (если есть) + total. Если корзина пуста — total = 0, items = [], всё равно создастся заявка с параметрами подбора.
